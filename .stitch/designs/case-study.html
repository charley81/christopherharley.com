A DJ Without a Stage

Merge Konflict had no website — only scattered social media profiles. For a working DJ, that’s a missed booking opportunity. Personal brand sites increase inquiries by ~40% compared to social‑media‑only presences.

In today's day and age in the underground scene, discoverability depends on owning a credible, searchable web presence. Transitioning from fragmented social handles to a unified platform was critical for professional expansion.

Designer, Developer, and Agent Wrangler

Solo Designer, Developer & Project Manager. My team was a fleet of specialized AI agents — Gemini CLI for implementation, sub‐agents for frontend, CMS, and code review, plus MCP integrations.

I planned the architecture, wrote the specs, reviewed every line, and steered the agents through each phase.

Spec‑Driven Agentic Engineering

The methodology followed a tight loop: Plan → Spec → Agent → Review → Merge → Deploy. This rigorous cycle ensured that every component met strict technical and design standards without manual coding.

01

Architecture Definition

Mapping out the single-page structure and CMS requirements before a single line of code is generated.

02

Precision Spec Writing

Creating high-fidelity prompts and Markdown specs that describe the UI and logic in exhaustive detail.

03

Agent Invocation

Deploying LLM agents via CLI to execute specific tasks based on the provided specs.

04

Human Review Loop

Validating the agent output against the design system and functional requirements.

05

Iterative Refinement

Providing corrective feedback to agents for edge cases, performance bugs, or styling tweaks.

06

Automated Deployment

Merging verified code into the main branch and triggering CI/CD pipelines to the edge.
